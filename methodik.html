Methodik & konzeptioneller Rahmen

Diese Seite ordnet das Projekt theoretisch ein. Sie stellt die zentrale Leitfrage vor und erklärt, auf welchen grundlegenden Konzepten und Forschungsfragen die Arbeit aufbaut.

Zentrale Begriffe

Die folgenden Begriffe sind zentral für das Projekt. Sie bilden die Grundlage für die Leitfrage und die beiden Teilfragestellungen.

Direct Manipulation

Direct Manipulation beschreibt eine Interaktionsform, bei der Nutzer direkt mit sichtbaren Objekten im Interface arbeiten. Aktionen wie Anklicken, Ziehen oder Ablegen führen sofort zu sichtbaren Ergebnissen.

Wichtig dabei ist, dass Nutzer jederzeit sehen, was im System passiert, und dass Aktionen meist einfach rückgängig gemacht werden können. Dadurch bleibt die Kontrolle klar beim Nutzer.[1]

Interface Agent

Ein Interface Agent ist eine softwarebasierte Komponente, die Nutzer bei ihrer Arbeit unterstützt. Er kann zum Beispiel Hinweise geben, Vorschläge machen oder Aktionen vorbereiten.

Im Unterschied zur Direct Manipulation handelt ein Agent teilweise eigenständig. Entscheidend ist dabei, dass Nutzer verstehen, was der Agent tut, und jederzeit eingreifen oder Entscheidungen selbst treffen können.[2][3]

Antwort auf die zentrale Leitfrage

Proaktive Unterstützung durch einen Interface Agent ist dann sinnvoll, wenn sie Nutzern Orientierung gibt und Sicherheit vermittelt, ohne ihnen die Kontrolle über das Interface zu nehmen. Gerade in Direct-Manipulation-Umgebungen bleibt die direkte Interaktion mit sichtbaren Objekten ein zentraler Bestandteil.[1]

1) Proaktiv, aber kontrollierbar

Automatisierung kann hilfreich sein, wird jedoch problematisch, wenn unklar ist, wer Entscheidungen trifft. Forschung zu Human Agency zeigt, dass Nutzer Unterstützung eher akzeptieren, wenn sie Vorschläge erhalten und Entscheidungen nachvollziehbar selbst treffen können.[4][5] Für dieses Projekt bedeutet das: Der Agent soll Optionen aufzeigen, nicht eigenständig handeln.

2) Verständlich und transparent

Damit proaktive Unterstützung akzeptiert wird, müssen Nutzer verstehen, was der Agent kann und warum er etwas vorschlägt. Leitlinien für Human-AI-Interaction empfehlen, Fähigkeiten und Grenzen des Systems klar zu kommunizieren und einfache Korrekturen zu ermöglichen.[6]

3) Unterstützung beim Wiedereinstieg

Nach Nutzungspausen verlieren Nutzer häufig die Orientierung. Bekannte Funktionen werden nicht sofort wieder erkannt.[7] Die Literatur zeigt, dass klare Navigation, sichtbare Einstiegspunkte, Verlaufsinformationen und visuelle Hinweise den Wiedereinstieg erleichtern können.[7][8][9][10] Der Agent unterstützt hier, indem er Kontext sichtbar macht, statt im Hintergrund Aktionen auszuführen.

Zusammengefasst kann ein Interface Agent Nutzer in Direct-Manipulation-Umgebungen sinnvoll unterstützen, wenn er Vorschläge macht, transparent bleibt und den Wiedereinstieg erleichtert. Die direkte Kontrolle über das Interface bleibt dabei stets beim Nutzer.[1][3]

Annahmen und Zielsetzung

Annahmen

  • Unterstützung wird eher akzeptiert, wenn Nutzer die Kontrolle behalten.
  • Unklare Automatisierung kann zu Unsicherheit führen.
  • Sichtbare Hinweise erleichtern den Wiedereinstieg nach Pausen.

Zielsetzung

Ziel des Projekts ist es, ein Interaktionskonzept und einen Prototyp zu entwickeln, der agentenbasierte Unterstützung bietet, ohne die Vorteile von Direct Manipulation zu verlieren.

Auf Basis dieser Überlegungen wurde ein Prototyp entwickelt, der unterschiedliche Formen agentenbasierter Unterstützung gegenüberstellt. So wird sichtbar, wie sich Kontrolle, Transparenz und Orientierung im Interface verändern.
