Interaktiver Prototyp

Der Prototyp (Figma Make) demonstriert drei Autonomiegrade von Interface Agents in einer Direct-Manipulation-Umgebung. Die Oberfläche bleibt bewusst gleich – nur das Verhalten des Systems ändert sich.

Wie man den Prototyp nutzt

Wechsle über den Slider zwischen No Agent, Over-Automated Agent und Assistive Agent und beobachte, wie sich Transparenz, Nachvollziehbarkeit und Kontrollgefühl verändern.

Die drei Modi im Überblick

Jeder Modus repräsentiert eine andere Agentenstrategie. Ziel ist ein klarer Vergleich von Unterstützung versus Kontrolle in einer Direct-Manipulation-Logik.

No Agent Default

Reine Direct Manipulation: Der Nutzer entscheidet und handelt vollständig selbst. Es gibt keine proaktive Unterstützung und keine impliziten Systementscheidungen.

  • Maximale Kontrolle
  • Hohe Nachvollziehbarkeit
  • Risiko: weniger Orientierung bei Unsicherheit

Over-Automated Agent

Das System trifft Entscheidungen implizit oder führt Aktionen aus, ohne diese klar zu erklären. Dieser Modus dient als Kontrastfall, um Risiken von Über-Automatisierung sichtbar zu machen.

  • Schnelle Ausführung
  • Geringere Transparenz
  • Risiko: Kontrollverlust / Unsicherheit

Assistive Agent

Unterstützt durch Erklärungen, Erinnerungen und Vorschläge – die Entscheidung bleibt beim Menschen. Fokus auf Transparenz, Eingriffsmöglichkeiten und Agency-Erhalt.

  • Hohe Transparenz (durch Erklärungen)
  • Nutzer entscheidet final
  • Risiko: Overload, wenn zu viele Unterstützungen getätigt werden

Vergleich der Modi

Die folgende Tabelle fasst die Unterschiede zusammen. Sie dient als schnelle Orientierung beim Testen.

Kriterium No Agent Over-Automated Agent Assistive Agent
Entscheidungsfindung Nutzer entscheidet Agent entscheidet implizit Nutzer entscheidet, Agent unterstützt
Transparenz hoch niedrig bis unklar hoch (durch Erklärungen)
Eingriffsmöglichkeiten jederzeit oft verspätet/unklar gezielt, bewusst angeboten
Unterbrechungen keine potenziell unerwartet kontextuell & steuerbar
Wahrgenommene Kontrolle (Agency) maximal reduziert erhalten/gestützt
Typisches Risiko fehlende Orientierung Kontrollverlust, Misstrauen zu viel Assistenz, wenn übertrieben

Design-Prinzipien (kompakt)

  • Erklären statt ungefragt ausführen
  • Vor Aktionen: ankündigen & um Zustimmung bitten
  • Undo / Rückgängig und Abbrechen anbieten
  • Klare Hinweise: „Was passiert?“ und „Warum passiert das?“
